[CRITICAL-01] Banca March peer shortfall formula is off by ~100x
- Evidence:
  - /Users/joaco/Spain funds market/pipeline/transformers/banca_march.py:157
  - Current code: implicit_revenue_shortfall_k = (median_bps - w_fee) / 100 * aum_k
  - This misses one /100 factor (bps -> decimal conversion requires /10,000).
- Impact:
  - Overstates revenue shortfall massively.
  - Current output shows March AM shortfall ~124,469k in /Users/joaco/Spain funds market/src/data/banca_march.json, when correct magnitude is ~1,245k.
  - This can drive materially wrong strategic recommendations.
- Recommendation:
  - Replace formula with: ((median_bps - w_fee) / 10000) * aum_k
  - Add unit-tested helper function for bps-to-decimal conversion used everywhere.
- Validator check to add:
  - Assert that shortfall for March AM ~= 1,244.5k for current snapshot.
